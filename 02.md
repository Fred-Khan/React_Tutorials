# PART 1: Break HTML into React Components

> **Goal**: Turn `<nav>`, `<main>`, and `<footer>` into reusable **components**.

### Step 2: Create Industry Standard Folder Structure

Let’s organise stuff like real-world apps:


### Step 3: Create `styles.css`

1. In `src/`, create a new file: `styles.css`
2. Paste **exactly** the CSS from your original `styles.css` (provided in the document):

```css
/* General Reset */

  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  font-family: Arial, sans-serif;
  line-height: 1.6;

}

/* Ensure full height layout */

  height: 100%;
  display: flex;
  flex-direction: column;
}

/* Let main content grow to fill space */
.content {
  flex: 1;
}

/* Navigation Bar */
.navbar {
  background-color: #007BFF;
  color: white;
  padding: 1rem 0;
}

.navbar .container {
  display: flex;
  justify-content: space-between;
  align-items: center;
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 1rem;
}

.navbar .logo {
  font-size: 1.5rem;
  font-weight: bold;
  text-decoration: none;
  color: white;
}

.navbar .nav-links {
  list-style: none;
  display: flex;
  gap: 1rem;
}

.navbar .nav-links li a {
  text-decoration: none;
  color: white;
  transition: color 0.3s;
}

.navbar .nav-links li a:hover {
  color: #FFD700;
}

/* Main Content */

  padding: 2rem 1rem;
  text-align: center;

  padding: 0 1rem;

```

> **Note**: We kept class names **exactly the same** (`navbar`, `content`, `footer`) so your HTML → React conversion is 1:1.

> **Note**: The footer CSS will have to be modified to ensure the footer is anchnored to the bottom of the page. A fix to this problem and the explanation has been added to the end of this tutorial.

### Step 4: Create Reusable Components

#### 1. `components/Navbar.tsx`

```tsx
// src/components/Navbar.tsx
import React from 'react';

const Navbar: React.FC = () => {
  return (
    <nav className="navbar">
      <div className="container">
        <a href="/" className="logo">MyWebsite</a>
        <ul className="nav-links">
          <li><a href="/">Home</a></li>
          <li><a href="/about">About</a></li>
          <li><a href="/login">Login</a></li>
        </ul>
      </div>
    </nav>
  );
};

export default Navbar;
```

> **Explanation**:
> - `React.FC` = **Function Component** with TypeScript typing
> - No props → `React.FC` is fine (we’ll use interfaces later)
> - `href="/"`, `/about`, etc. → will work with React Router later

---

#### 2. `components/Footer.tsx`

```tsx
// src/components/Footer.tsx
import React from 'react';

const Footer: React.FC = () => {
  return (
    <footer className="footer">
      <div className="container">
        <p>&copy; 2025 MyWebsite. All rights reserved.</p>
      </div>
    </footer>
  );
};

export default Footer;
```

---

#### 3. `components/Layout.tsx` (Shared Layout)

This is a **best practice**: wrap pages with common structure.

```tsx
// src/components/Layout.tsx
import React from 'react';
import Navbar from './Navbar';
import Footer from './Footer';

interface LayoutProps {
  children: React.ReactNode;
}

const Layout: React.FC<LayoutProps> = ({ children }) => {
  return (
    <>
      <Navbar />
      <main className="content">{children}</main>
      <Footer />
    </>
  );
};

export default Layout;
```

> **New Concept**: `children` prop  
> - Lets you **inject** page content between Navbar and Footer  
> - Like `<slot>` in Vue or Web Components

# PART 2: Apply Global CSS

# PART 3: Add Routing with React Router v6

> **Goal**: Click “About” → go to `/about` with same navbar/footer

---

### Step 1: Install React Router

```bash
npm install react-router-dom
```

> **Note**: Vite + TypeScript needs types:
```bash
npm install --save-dev @types/react-router-dom
```

---

### Step 2: Set Up Router in `main.tsx`

```tsx
// src/main.tsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import './styles.css';
import App from './App.tsx';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
);
```

> **Explanation**:  
> `BrowserRouter` enables client-side routing (no page reloads)

---

### Step 3: Define Routes in `App.tsx`

```tsx
// src/App.tsx
import { Routes, Route } from 'react-router-dom';
import Layout from './components/Layout';
import Home from './pages/Home';
import About from './pages/About';

function App() {
  return (
    <Routes>
      <Route element={<Layout />}>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        {/* Future: <Route path="/login" element={<Login />} /> */}
      </Route>
    </Routes>
  );
}

export default App;
```

> **New Concept**: **Nested Routes**  
> - `<Route element={<Layout />}>` = all child routes get Navbar + Footer  
> - `path="/"` = Home page  
> - `element={<Home />}` = what to render

---

### Step 4: Update `Layout.tsx` to Use `<Outlet />` to clear the error in app.tsx

```markdown
> **Important Fix**: We originally passed `children` manually.  
> But with **React Router**, the correct way is to use `<Outlet />`.  
> This is the **industry standard** and avoids TypeScript errors.
```


```tsx
// src/components/Layout.tsx
import React from 'react';
import { Outlet } from 'react-router-dom'; // ← Add this import
import Navbar from './Navbar';
import Footer from './Footer';

const Layout: React.FC = () => {
  return (
    <>
      <Navbar />
      <main className="content">
        <Outlet />  {/* ← This renders the current page */}
      </main>
      <Footer />
    </>
  );
};

export default Layout;
```

Update `Navbar.tsx`:

```tsx
import React from 'react';

const Navbar: React.FC = () => {
  return (
    <nav className="navbar">
      <div className="container">

        <ul className="nav-links">
```

> - Prevents full page reload  
> - Faster, smoother UX  
> - **Industry standard**

### Fix Footer Not Sticking to Bottom!!!

When content is short, the footer appears right under the text.  
**Why?** React renders inside `<div id="root">`, breaking the original `html, body` flex.

**Solution** – Wrap the app in a flex container:

Below is a fix that will make the **footer stick to the bottom of the viewport** even when the page content is short (like the About page).  
I’ll show you **exactly what to change**, **why it works**, and give you a **copy-paste ready patch**.

---

## The Problem (in plain English)

Your original CSS used:

```css
html, body { height: 100%; display: flex; flex-direction: column; }
.content { flex: 1; }
```

In a **plain HTML file** this works because the `<body>` contains **one** flex container.  
In **React + Vite** the root `<div id="root">` is *inside* the `<body>`, so the flex rules never see the full height → the footer floats right under the content.

---

## The Fix – Wrap Everything in a Flex Container

We’ll add a **wrapper div** around the whole app (inside `main.tsx`) and move the flex styles there.

### 1. Update `src/main.tsx`

```tsx
// src/main.tsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import './styles.css';
import App from './App.tsx';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      {/* NEW: Flex wrapper */}
      <div className="app-wrapper">
        <App />
      </div>
    </BrowserRouter>
  </StrictMode>
);
```

---

### 2. Add the wrapper styles to `src/styles.css`

Add **these lines** **anywhere** in `styles.css` (I put them right after the `body` rule for clarity):

```css
/* Flex wrapper to push footer to bottom */
.app-wrapper {
  min-height: 100vh;           /* Full viewport height */
  display: flex;
  flex-direction: column;
}

/* Let main content grow to fill space */
.content {
  flex: 1;                     /* Already exists – keep it! */
}
```

> **Note**: `min-height: 100vh` works even if the browser window is resized.

---

### 3. Clean up the old `html, body` rule

You can safely **delete** these lines from `styles.css` (they’re no longer needed):

```css
/* Ensure full height layout */
html, body {
  height: 100%;
  display: flex;
  flex-direction: column;
}
```

Leave the rest untouched.

---

|------|-----------------|
| **Home** (tall content) | At bottom of content |
| **About** (short content) | **Stays at bottom of screen** |

No more “floating” footer!

# Best Practices for Beginners

| Practice | Why It Matters |
|--------|----------------|
| **Use `React.FC` or explicit props interface** | Type safety → fewer bugs |
| **Keep components small & reusable** | Easier to debug and reuse |
| **Use `Layout` for shared UI** | Avoid duplicating Navbar/Footer |
| **Prefer `Link` over `<a>`** | Better performance |
| **Name files `.tsx` for components** | Clarity |
| **Use meaningful folder structure** | Scales with project size |

# Important Information: CSS Modules vs Global CSS

### Global CSS (What We Used)
```css
/* styles.css */
.navbar { background: blue; }
```
```tsx
<nav className="navbar">...</nav>
```

**Pros**:
- Simple, familiar
- One file for all styles

**Cons**:
- Risk of **class name collisions**
- Hard to remove unused CSS

---

### CSS Modules (Alternative)

1. Rename: `Navbar.module.css`
2. Write:
   ```css
   .navbar { background: blue; }
   ```
3. Use:
   ```tsx
   import styles from './Navbar.module.css';
   <nav className={styles.navbar}>
   ```

**Pros**:
- **Scoped class names** → `navbar` becomes `_navbar_abc123`
- No collisions
- Unused styles removed automatically

**Cons**:
- Slightly more setup
- Can’t use global selectors easily (e.g., `body`, `html`)

> **My Recommendation**: Use **global CSS for layouts**, **CSS Modules for components**
